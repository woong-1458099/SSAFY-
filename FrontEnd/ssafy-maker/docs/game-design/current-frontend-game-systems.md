# 현재 프론트 구현 게임 시스템 정리

- 상태: 구현 기준 정리
- 기준일: 2026-03-25
- 범위: `src/` 기준 프론트에 실제 연결된 게임 시스템만 정리했다. 미사용 에셋이나 폐기된 레거시는 제외했다.

## 1. 현재 게임의 핵심 루프

현재 프론트의 메인 루프는 아래 흐름으로 구성된다.

1. 로그인/세션 확인
2. 시작 화면에서 새 게임 또는 이어하기 선택
3. 새 게임이면 인트로 후 캐릭터 생성
4. 메인 월드에서 이동, NPC 대화, 장소 이용, 주간 계획, 미니게임, 고정 이벤트 진행
5. 시간대와 주차가 진행되며 스탯/체력/스트레스/재화가 누적 변화
6. 6주차 종료 후 엔딩 판정 및 엔딩 연출 재생

한 줄로 요약하면:

`주간 계획 + 행동력 소모 + 장소 액션 + 미니게임 + 고정 이벤트 + 스탯 누적`이 현재 구현된 핵심 플레이 구조다.

## 2. 구현된 게임 시스템 목록

### 2.1 인증 / 세션 진입 시스템

- 로그인/회원가입 UI가 구현되어 있다.
- BFF 세션 기반 인증 흐름을 사용한다.
- 개발용 인증 우회 입장 버튼도 있다.
- 세션이 살아 있으면 로그인 화면을 건너뛰고 시작 화면으로 진입한다.
- 로그아웃 기능이 시작 화면과 인게임 메뉴에 연결되어 있다.

### 2.2 시작 화면 / 이어하기 시스템

- 새 게임 시작 버튼이 있다.
- 저장 슬롯이 있으면 이어하기 버튼이 활성화된다.
- 이어하기 모달에서 저장 슬롯을 선택해 즉시 메인 씬으로 복귀할 수 있다.
- 저장 데이터 기준으로 지역, 씬, 플레이어 위치, 진행 상태를 복원한다.

### 2.3 캐릭터 생성 시스템

- 이름 입력
- 성별 선택
- 헤어 선택
- 의상 선택
- 선택 결과를 플레이어 외형 데이터로 저장

캐릭터 생성 완료 시 `playerData`와 `isNewCharacter`가 등록되고 메인 월드로 진입한다.

### 2.4 월드 탐색 / 지역 이동 시스템

- 메인 월드 씬은 TMX 타일맵 기반으로 동작한다.
- 지역은 `전체 지도(world)`, `캠퍼스(campus)`, `번화가(downtown)` 3개가 구현되어 있다.
- 플레이어 이동, 충돌 처리, 상호작용 레이어, 전경 레이어, 지역 전환 오버레이가 있다.
- 지역 전환은 월드 포털/트랜지션 타일 위에서 `SPACE` 상호작용으로 처리된다.

현재 구조상 월드 맵은 허브 역할에 가깝고, 실제 보상/리소스 소비는 장소 액션과 이벤트에서 발생한다.

### 2.5 NPC 상호작용 / 대화 시스템

- 근접 NPC 감지
- `SPACE` 대화 상호작용
- 선택지형 대화
- 선택지 조건 체크
  - FE / BE / 협업 / 운 / 스트레스
  - HP / 돈
  - 플레이어 성별
  - NPC 호감도
- 대화 결과 적용
  - 스탯 변화
  - 호감도 변화
  - 플래그 설정
  - HUD 값 변경
  - 장소/미니게임 액션 실행

즉, 대화는 단순 텍스트가 아니라 스탯/호감도/행동 분기 시스템으로 연결되어 있다.

### 2.6 플레이어 상태 / 성장 시스템

현재 프론트에서 관리하는 주요 상태는 다음과 같다.

- 기본 스탯: `FE`, `BE`, `협업(teamwork)`, `운(luck)`, `스트레스(stress)`
- HUD 상태: `시간대`, `현재 위치`, `주차`, `요일`, `행동력`, `최대 행동력`, `HP`, `최대 HP`, `돈`
- 추가 상태: `호감도`, `플래그`

기본 시작값도 프론트에 정의되어 있으며, 대부분의 시스템이 이 상태를 직접 갱신한다.

### 2.7 시간 / 주차 / 행동력 진행 시스템

- 시간대: `오전`, `오후`, `저녁`, `밤`
- 요일: `월요일` ~ `일요일`
- 기본 행동력: 4
- 행동력 소모 시 시간대가 전진한다.
- 밤 이후 다음 날로 넘어가면 행동력이 회복된다.
- 주차는 1주차부터 시작하며 6주차 종료 후 엔딩 플로우로 넘어간다.
- 주급 지급 시스템이 있다.
  - 주 시작 시 `50,000G` 지급
- 특정 시간에 고정 이벤트가 남아 있으면 시간 진행이 차단된다.

이 시스템이 현재 게임 전체의 턴 구조를 담당한다.

### 2.8 주간 계획 시스템

- 주 시작 시 주간 계획표 모달이 열린다.
- 평일 오전/오후 슬롯 기준으로 계획을 지정한다.
- 현재 선택 가능한 계획은 3종이다.
  - `UI 구현 연습`: FE +2
  - `REST API와 데이터베이스 설계`: BE +2
  - `팀 프로젝트`: 협업 +2
- 해당 시간 슬롯을 실제로 넘길 때 계획 보상이 지급된다.
- 고정 이벤트가 있는 슬롯은 계획표에 배지로 표시된다.

즉, 주간 계획은 현재 게임에서 가장 명확한 중장기 성장 예약 시스템이다.

### 2.9 장소 액션 시스템

정적 장소에 접근하면 설명 모달 또는 액션 모달이 열린다.

현재 구현된 장소와 효과는 다음과 같다.

- `집(home)`
  - 잠자기: HP 회복, 스트레스 감소
  - 공부하기: FE/BE 증가, HP 감소, 스트레스 증가
  - 게임하기: 스트레스 감소, 운 증가
- `편의점(store)`
  - 상점 열기
- `카페(cafe)`
  - 돈 소비, HP 회복, 스트레스 감소
- `헬스장(gym)`
  - 돈 소비, 최대 HP 증가, 협업 증가, 미니게임 실행
- `라멘띵스(ramen)`
  - 돈 소비, HP 회복, 스트레스 감소, 미니게임 실행
- `복권 판매점(lotto)`
  - 돈 소비, 로또 미니게임 실행
- `노래방(karaoke)`
  - 돈 소비, 스트레스 감소, 협업 증가, 미니게임 실행
- `역전할머니호프(beer)`
  - 스트레스 감소, 협업 증가, 돈 획득, 미니게임 실행

이용 제한도 있다.

- 카페는 밤 이용 불가
- 헬스장/라멘/로또는 밤 이용 불가
- 호프는 저녁/밤에만 이용 가능

### 2.10 미니게임 시스템

미니게임 메뉴 씬과 개별 미니게임 씬이 연결되어 있다.

현재 활성 미니게임 목록:

- Quiz
- Rhythm
- Interview
- Runner
- Tank
- Business Smile
- Don't Smile
- Gym
- Cooking
- Lotto
- Drinking

현재 비활성/제외 상태:

- Typing: 비활성화
- Drag 계열: deprecated

접근 경로는 3가지다.

- 장소 액션에서 직접 실행
- 대화 액션에서 실행
- 미니게임 메뉴에서 선택 실행

### 2.11 미니게임 보상 / 결과 반영 시스템

- 미니게임 종료 후 메인 씬으로 복귀하는 구조가 있다.
- 일부 장소는 장소 효과를 먼저 적용하고 미니게임을 연다.
- 로또는 별도 완료 이벤트를 통해 금전 보상을 메인 HUD에 반영한다.
- 카탈로그에는 미니게임별 의도 보상 문구가 정의되어 있다.

다만 현재 보상 구조는 미니게임별로 완전 통합된 공용 메타 시스템이라기보다, 개별 미니게임/장소 효과 기반 반영에 가깝다.

### 2.12 고정 이벤트 / 로맨스 이벤트 시스템

- 주차별 고정 이벤트 JSON 로드
- 현재 주차 기준 이벤트 데이터 비동기 로드
- 특정 요일/시간/장소에 맞는 이벤트 자동 판정
- 이벤트 진행 후 완료 처리
- 완료된 고정 이벤트는 재실행 방지
- 이벤트 진행 후 시간 자동 소모
- 주간 계획표 슬롯에 이벤트 위치 표시
- 디버그 기준으로 1주차부터 6주차까지 주별 데이터 관리
- 로맨스 이벤트는 플레이어 성별에 따라 분기 필터링된다.

현재 스토리의 실질적인 강제 진행 장치는 이 고정 이벤트 시스템이다.

## 4. 현재 프론트 기준 핵심성 정리

현재 구현만 놓고 보면, 이 게임의 핵심 시스템 후보는 이미 아래 조합으로 형성되어 있다.

- `주간 계획표`로 성장 방향을 예약한다.
- `행동력/시간`을 소모하며 하루를 운영한다.
- `장소 액션`과 `미니게임`으로 즉시 보상을 얻는다.
- `고정 이벤트`가 특정 시간대를 강제 점유하며 서사를 밀어 준다.
- 누적된 `스탯/체력/운/협업`이 엔딩을 결정한다.

반대로 현재 상대적으로 약한 부분은 아래다.

- 월드 탐색 자체의 고유 보상
- 장기적으로 반복 플레이를 끄는 고유 메타 목표
- 장소별 독자 규칙이 쌓이는 심화 시스템
- 미니게임 성공/실패가 장기 구조에 미치는 차별화

따라서 “현재 구현된 게임의 실제 중심”은 탐험형 RPG라기보다, `주간 관리형 육성 시뮬레이션 + 장소 액션 + 미니게임 번들 + 고정 이벤트 서사`에 가깝다.
