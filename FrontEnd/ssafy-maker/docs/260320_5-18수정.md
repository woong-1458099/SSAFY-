## 변경 묶음별 설명

### 1. 고정 이벤트 데이터
- 파일:
  - `public/assets/game/data/story/fixedevent/fixed_week1.json`
  - `public/assets/game/data/story/fixedevent/fixed_week2.json`
  - `public/assets/game/data/story/fixedevent/fixed_week3.json`
  - `public/assets/game/data/story/fixedevent/fixed_week4.json`
  - `public/assets/game/data/story/fixedevent/fixed_week5.json`
  - `public/assets/game/data/story/fixedevent/fixed_week6.json`
- 바뀐 방향:
  - 주차별 이벤트 대사/분기 내용 손봄
  - 재화 조건과 재화 소모량을 현재 경제 스케일에 맞게 상향
  - 예전 `3000 / 10000 / 30000` 계열 금액이 더 큰 단위로 올라감
- 이해 포인트:
  - 이 JSON들은 “언제 어떤 이벤트가 뜨고, 선택지 결과가 뭔지”를 들고 있는 데이터 파일
  - 코드보다 먼저 이 데이터가 바뀌면 실제 플레이 체감이 크게 바뀜

### 2. 자취방/번화가/소비템 밸런스
- 파일:
  - `src/features/home/homeActions.ts`
  - `src/features/place/placeActions.ts`
  - `src/scenes/MainScene.ts`
- 바뀐 방향:
  - 자취방 행동 수치 재조정
  - 번화가 가게별 이용료/효과 재조정
  - 편의점 소비템 가격/판매가/효과 재조정
- 이해 포인트:
  - `homeActions.ts`는 자취방 행동 공식
  - `placeActions.ts`는 번화가/카페 공식
  - `MainScene.ts` 안 `SHOP_ITEM_TEMPLATES`는 편의점 아이템 공식
- 의도:
  - 자취방 = 돈 안 들고 시간 1칸 쓰는 기본 행동
  - 번화가 = 돈+시간이 드는 대신 성격이 분명한 행동
  - 소비템 = 시간은 안 쓰지만 더 비싸고 효율은 낮은 응급 행동

### 3. 경제 스케일 상향
- 파일:
  - `src/scenes/MainScene.ts`
  - `src/features/ui/components/game-hud.ts`
  - `src/features/place/placeActions.ts`
  - `src/features/minigame/lottoOutcome.ts`
  - `public/assets/game/data/story/fixedevent/*.json`
- 바뀐 방향:
  - 시작 소지금 상향
  - 전체 소비 가격 상향
  - 로또 가격/당첨금 상향
  - 스토리 재화 분기 상향
- 이해 포인트:
  - 월급만 올리면 물가가 그대로라 밸런스가 무너짐
  - 그래서 생활비, 가게 비용, 복권, 이벤트 재화 조건을 같이 올린 구조

### 4. 월요일 주급 지급
- 파일:
  - `src/scenes/MainScene.ts`
  - `src/features/save/saveGameState.ts`
- 바뀐 방향:
  - 매주 월요일 주급 지급
  - 지급 주차 저장
  - 저장/불러오기 후 중복 지급 방지
- 흐름:
  - 월요일 진입
  - 주급 지급
  - 주급 팝업 표시
  - 팝업 닫으면 주간계획표
  - 그 다음 주간 활동/고정 이벤트
- 이해 포인트:
  - `MainScene.ts`가 실제 순서를 제어
  - `saveGameState.ts`는 저장 메타 문자열과 저장 구조 담당

### 5. 로또 구조 변경
- 파일:
  - `src/features/minigame/lottoOutcome.ts`
  - `src/scenes/legacyMinigames/LottoScene.ts`
  - `src/scenes/MainScene.ts`
- 바뀐 방향:
  - 기존 즉시 랜덤 지급이 아니라 미니게임 결과 중심으로 정리
  - 로또 보상표 별도 분리
  - 로또 가격/당첨금이 새 경제 스케일에 맞게 올라감
- 이해 포인트:
  - `lottoOutcome.ts` = 확률표/상금표
  - `LottoScene.ts` = 실제 로또 미니게임 화면
  - `MainScene.ts` = 결과를 돈으로 반영하고 엔딩 연결

### 6. NPC 배치/표시 구조 변경

  - 라벨 색/위치 조정
  - 건물 상호작용 하이라이트 방식 조정
  - 상시 보상용 NPC 일부 제거
- 이해 포인트:
  - `npcPositions.ts` = 맵에 늘 서 있는 NPC 기본 좌표
  - `fixedEventNpcPresence.ts` = 특정 이벤트 시간대에만 뜨는 NPC 좌표/표시 정책
