# SSAFY Maker 시연 테스트 시나리오

## 1. 문서 목적

이 문서는 `S14P21E206` 프로젝트의 시연 시, 발표자 또는 평가자가 현재 구현된 게임 흐름을 순서대로 확인할 수 있도록 만든 간단한 시연 절차서다.

현재 저장소 기준 주요 런타임 흐름은 아래 순서다.

1. `BootScene`
2. `PreloadScene`
3. `LoginScene`
4. `StartScene`
5. `IntroScene`
6. `NewCharacterScene`
7. `MainScene`
8. `FinalSummaryScene`
9. `EndingIntroScene`
10. `EndingComicScene`
11. `StartScene` 복귀

## 2. 시연 환경

- 시연 URL
  - STG 기준: `https://stg.ssafymaker.cloud`
  - 운영 기준: `https://ssafymaker.cloud`
- 권장 브라우저
  - Chrome 최신 버전
- 권장 해상도
  - FHD 이상
- 준비 상태
  - 프론트엔드, 백엔드, Keycloak 정상 기동
  - DB 덤프 복원 완료
  - 테스트 계정 또는 개발용 bypass 로그인 사용 가능 상태

## 3. 사전 준비 체크

- 브라우저 캐시를 초기화하거나 시크릿 창으로 접속한다.
- 로그인 계정이 준비되어 있는지 확인한다.
- 시연 중 저장/불러오기 테스트를 위해 최소 1개의 저장 슬롯이 있거나 새로 저장할 수 있는 상태인지 확인한다.
- 엔딩 시연을 짧게 진행해야 하는 경우, 사전에 엔딩 직전 저장 데이터를 준비해 둔다.

## 4. 시연 시나리오

### 4.1 서비스 진입 및 로그인 화면 확인

- 시작 위치
  - 브라우저에서 시연 URL 접속
- 수행 동작
  - 메인 페이지가 로드될 때까지 대기
- 확인 포인트
  - 로그인/회원가입 UI가 보인다.
  - 좌측 패널에 최근 사망 기록과 사망 랭킹 영역이 보인다.
  - 로그인 버튼과 회원가입 탭이 보인다.

### 4.2 로그인 수행

- 시작 위치
  - `LoginScene`
- 수행 동작
  - 기본 시연:
    - `로그인` 버튼 클릭
    - Keycloak 로그인 화면에서 테스트 계정 입력 후 로그인
  - 대체 시연:
    - 개발 환경에서만 `[ 개발용 ] 백엔드 없이 바로 접속하기` 버튼 클릭
- 확인 포인트
  - 인증 성공 후 시작 화면으로 이동한다.
  - 로그인 실패 시 오류 메시지 또는 재시도 가능한 화면이 유지된다.

### 4.3 시작 화면 확인

- 시작 위치
  - `StartScene`
- 수행 동작
  - 화면의 `새 게임`, `이어하기`, `로그아웃` 요소 확인
- 확인 포인트
  - `새 게임` 버튼이 노출된다.
  - 저장 데이터가 있으면 `이어하기` 버튼이 활성화된다.
  - 우측 상단 또는 상단 영역에 `로그아웃` 버튼이 보인다.

### 4.4 새 게임 시작

- 시작 위치
  - `StartScene`
- 수행 동작
  - `새 게임` 버튼 클릭
- 확인 포인트
  - 인트로 화면으로 전환된다.
  - 화면 전환이 자연스럽게 이어진다.

### 4.5 인트로 스킵 또는 진행

- 시작 위치
  - `IntroScene`
- 수행 동작
  - 인트로를 짧게 본 뒤 진행
  - 필요 시 화면 클릭으로 스킵
- 확인 포인트
  - 인트로가 정상 재생된다.
  - 스킵 안내가 보인다.
  - 다음 캐릭터 생성 화면으로 전환된다.

### 4.6 캐릭터 생성

- 시작 위치
  - `NewCharacterScene`
- 수행 동작
  - 이름 영역 클릭 후 임의 이름 입력
  - 남/여 캐릭터 버튼 클릭
  - 헤어 번호 선택
  - 의상 번호 선택
  - `생성하기` 버튼 클릭
- 확인 포인트
  - 선택한 성별, 헤어, 의상에 따라 캐릭터 미리보기가 바뀐다.
  - 이름 변경이 반영된다.
  - 생성 후 메인 게임 화면으로 진입한다.

### 4.7 메인 게임 진입 확인

- 시작 위치
  - `MainScene`
- 수행 동작
  - 플레이어가 위치한 메인 화면 확인
  - HUD, 메뉴, 배경, 플레이어 캐릭터 확인
- 확인 포인트
  - 게임 월드가 정상 노출된다.
  - UI/HUD가 함께 보인다.
  - 캐릭터 이동 또는 상호작용이 가능하다.

### 4.8 기본 조작 시연

- 시작 위치
  - `MainScene`
- 수행 동작
  - `WASD` 또는 방향키로 캐릭터 이동
  - `Space` 키로 상호작용 시도
  - `P` 키로 일정표 또는 주간 계획 UI 호출
  - `ESC` 키로 메뉴 호출
- 확인 포인트
  - 이동 입력이 정상 반영된다.
  - 상호작용 가능한 오브젝트 또는 장소에서 UI가 반응한다.
  - 일정표, 메뉴 UI가 정상 열리고 닫힌다.

### 4.9 일정표/행동 선택

- 시작 위치
  - `MainScene`
- 수행 동작
  - `P` 키로 일정표 오픈
  - 강의 또는 행동 하나를 선택
- 확인 포인트
  - 행동 선택 UI가 정상 표시된다.
  - 선택 후 시간, 주차, 스탯 또는 진행 상태가 변한다.

### 4.10 저장 기능 시연

- 시작 위치
  - `MainScene`
- 수행 동작
  - `ESC` 키로 메뉴 열기
  - 저장 관련 메뉴 진입
  - 비어 있는 슬롯 또는 원하는 슬롯에 저장
- 확인 포인트
  - 저장 슬롯 목록이 표시된다.
  - 저장 완료 후 슬롯 메타 정보가 반영된다.
  - 이후 시작 화면의 `이어하기`에 반영될 수 있다.

### 4.11 이어하기 기능 시연

- 시작 위치
  - `StartScene`
- 수행 동작
  - 로그아웃하지 않은 상태 또는 재진입 후 `이어하기` 버튼 클릭
  - 저장 슬롯 하나 선택
- 확인 포인트
  - 슬롯 목록이 팝업으로 보인다.
  - 선택한 저장 슬롯 기준으로 게임이 복원된다.

### 4.12 사망 기록/랭킹 시연

- 시작 위치
  - `LoginScene`
- 수행 동작
  - 로그인 화면 좌측 패널 확인
- 확인 포인트
  - 최근 사망 기록 목록이 보인다.
  - 사망 랭킹 목록이 보인다.
  - 데이터가 없는 경우에도 빈 상태 문구가 정상 표시된다.

### 4.13 엔딩 흐름 시연

- 시작 위치
  - 엔딩 직전 저장 데이터 또는 충분히 진행된 `MainScene`
- 수행 동작
  - 주차를 끝까지 진행하거나 준비된 세이브를 불러와 엔딩 조건을 만족시킨다.
- 확인 포인트
  - `FinalSummaryScene` 진입
  - `EndingIntroScene` 진입
  - `EndingComicScene` 진입
  - 엔딩 종료 후 시작 화면으로 돌아간다.

## 5. 권장 시연 구성

### 5.1 최소 시연 구성

짧은 발표에서는 아래 흐름만 보여도 기본 구조 설명이 가능하다.

1. 로그인 화면 진입
2. 로그인
3. 시작 화면
4. 새 게임
5. 캐릭터 생성
6. 메인 게임 진입
7. 일정표 또는 메뉴 호출
8. 저장 시연

### 5.2 확장 시연 구성

시간이 충분하면 아래까지 포함한다.

1. 사망 기록/랭킹 패널 설명
2. 이어하기
3. 저장 데이터 복원
4. 엔딩 장면

## 6. 시연 중 자주 쓰는 대사 포인트

- 로그인 화면에서는 Keycloak 기반 인증과 공개 사망 통계 패널을 함께 보여줄 수 있다.
- 시작 화면에서는 새 게임/이어하기/로그아웃 흐름을 설명할 수 있다.
- 캐릭터 생성 화면에서는 사용자 커스터마이징 요소를 설명할 수 있다.
- 메인 게임에서는 이동, 일정표, 상호작용, 저장 기능을 중심으로 설명하면 된다.
- 엔딩 시연은 시간이 오래 걸릴 수 있으므로 별도 저장 데이터를 준비한 뒤 보여주는 방식이 안전하다.

## 7. 예외 대응 시나리오

### 7.1 로그인 지연 또는 인증 서버 문제

- Keycloak 응답이 느리면 잠시 대기 후 재시도한다.
- 개발 환경에서는 bypass 로그인으로 대체 시연한다.

### 7.2 저장 데이터가 없는 경우

- 먼저 새 게임으로 진입해 간단한 저장을 1회 만든 뒤 `이어하기`를 다시 보여준다.

### 7.3 엔딩까지 시간이 오래 걸리는 경우

- 엔딩 직전 저장 데이터를 미리 준비해 불러오기 후 엔딩만 시연한다.

## 8. 문서 한계

- 본 문서는 현재 저장소 구현 기준의 시연용 초안이다.
- 실제 발표 직전에는 운영 배포본 기준 화면 문구, 버튼 위치, 테스트 계정, 저장 슬롯 상태를 다시 확인해 최종본으로 정리해야 한다.
