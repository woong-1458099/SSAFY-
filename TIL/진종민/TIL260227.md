# TIL — 2026-02-27 (Unity 첫 세팅 및 기초 구조 잡기)

## 개요

오늘은 Unity를 처음 설치하고, **도트형 캐주얼 2D 게임 프로젝트(싸피생 키우기)**를 만들기 위한 기본 환경 설정과 구조 이해를 진행했다.

---

## 오늘 한 작업

### 1. Unity 버전 및 템플릿 선택

* **Unity 6.3 LTS** 선택
* 새 프로젝트이기 때문에 최신 안정 버전으로 시작
* 프로젝트 성격에 맞춰 **2D (Built-In Render Pipeline)** 템플릿 사용

### 2. 프로젝트 생성

* 프로젝트 생성 후 Unity 기본 화면 구성 확인
* 각 창의 역할 파악

  * **Hierarchy** : 현재 씬의 오브젝트 목록
  * **Scene** : 오브젝트 배치 공간
  * **Game** : 실제 플레이 화면
  * **Inspector** : 오브젝트 속성 설정
  * **Project** : 에셋 및 파일 관리 공간

### 3. 기본 폴더 구조 생성

프로젝트 정리를 위해 아래 폴더를 생성했다.

* `Scenes`
* `Scripts`
* `Sprites`
* `Prefabs`
* `UI`

### 4. UI 작업 준비

* **Canvas** 생성
* TextMeshPro를 이용한 텍스트 UI 준비
* 버튼 UI를 배치하여 육성형 게임의 기본 인터페이스 구성을 시작함

### 5. GameManager 오브젝트 및 스크립트 생성

* `GameManager` 빈 오브젝트 생성
* `GameManager.cs` 스크립트 생성 후 오브젝트에 연결 시도

### 6. 스크립트 오류 해결

처음에는 스크립트가 오브젝트에 붙지 않는 문제가 발생했다.
원인은 Unity에서 오브젝트에 붙일 수 있는 스크립트는 반드시 `MonoBehaviour`를 상속해야 하기 때문이었다.

```csharp
using UnityEngine;

public class GameManager : MonoBehaviour
{
}
```

위와 같은 형태여야 Unity 컴포넌트로 정상 인식된다는 점을 확인했다.

### 7. 코드 편집기 관련 확인

* Unity에서 `.cs` 파일을 더블클릭하면 기본적으로 Visual Studio로 연결됨
* 하지만 **VS Code로도 외부 스크립트 편집기 변경 가능**하다는 점 확인

### 8. 한글 깨짐 문제 확인

* 코드에서는 한글이 정상
* 하지만 **Game 화면에서만 한글이 깨짐**
* 원인은 **TextMeshPro 기본 폰트가 한글을 지원하지 않기 때문**
* 추후 한글 지원 폰트를 추가하고 TMP Font Asset으로 설정해야 함

---

## 오늘 배운 점

* 새 Unity 프로젝트는 **Unity 6.3 LTS**로 시작하는 것이 안정적이다.
* 도트형 캐주얼 게임은 **2D (Built-In Render Pipeline)** 이 단순하고 시작하기 좋다.
* Unity는 에디터 창의 역할을 먼저 이해하는 것이 중요하다.
* UI는 **Canvas** 위에서 관리된다.
* Unity 오브젝트에 붙는 스크립트는 반드시 **`MonoBehaviour`를 상속**해야 한다.
* C# 스크립트는 Visual Studio뿐 아니라 **VS Code로도 연결 가능**하다.
* TextMeshPro는 기본 설정만으로 한글이 깨질 수 있어 **폰트 설정이 중요**하다.

---

## 느낀 점

Unity를 처음 사용해서 화면 구성부터 낯설었지만,
오늘은 무작정 기능 구현을 하기보다 **기본 세팅과 구조 이해, 오류 해결**에 집중했다.

특히 스크립트 연결 오류와 한글 폰트 깨짐 문제를 겪으면서,
Unity는 단순히 코드만 작성하는 환경이 아니라
**오브젝트-컴포넌트 구조와 에셋 설정까지 함께 이해해야 하는 도구**라는 점을 느꼈다.

오늘은 본격적인 기능 구현 전 단계였지만,
프로젝트의 기초를 잡았다는 점에서 의미 있는 시작이었다.
