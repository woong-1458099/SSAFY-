# TIL — 2026-02-26 (4-bit Quantization과 QLoRA)

## 오늘 한 일 요약
- 4-bit 양자화로 모델을 경량화하고 QLoRA로 파인튜닝하는 흐름을 정리했다.
- 학습 후 추론 및 평가 파이프라인을 확인했다.

## 핵심 배움
- 양자화는 성능을 최대한 유지하면서 모델 크기와 메모리를 줄이는 기법이다.
- QLoRA는 4-bit로 양자화된 모델에 LoRA를 적용해 저메모리 환경에서도 학습 가능하다.
- Text-to-SQL과 같은 특정 task에서는 전처리와 평가 지표 관리가 중요하다.

## 진행 순서
- `uv sync`로 가상환경과 의존성을 설치했다.
- `QLoRA.ipynb`에서 학습 셀을 순서대로 실행했다.
- `python inference.py`로 예측 결과를 생성했다.
- `test-suite-sql-eval-master`에서 평가 스크립트를 실행했다.

## 주요 파일/폴더
- `QLoRA.ipynb`: QLoRA 학습 노트북
- `inference.py`: 추론 스크립트
- `data/train.csv`, `data/validation.csv`: 학습/검증 데이터
- `test-suite-sql-eval-master/`: 평가 코드와 예시
- `outputs/`: 학습된 모델 저장 위치

## 사용한 명령어
- `uv sync`
- 필요 시 `uv add numpy==2.1.0 pandas==2.2.3 peft==0.17.1 torch==2.9.0 transformers==4.56.0 accelerate==1.10.1 datasets==4.0.0 trl==0.22.2 sqlparse==0.5.5 nltk==3.9.2 jupyter==1.1.1 ipykernel==7.1.0 bitsandbytes==0.49.1`
- `python -c "import nltk; nltk.download('punkt_tab')"`
- `cd test-suite-sql-eval-master; python evaluation.py --gold gold.txt --pred base_model_predict.txt --db database/ --etype exec --plug_value`

## 트러블슈팅 메모
- bitsandbytes 설치와 CUDA 버전 매칭이 핵심이다.
- 평가 전 `database/`에 DB 파일이 있어야 한다.

## 환경 요약
- Python 3.13
- CUDA 12.9.1
- GPU VRAM 6GB+
- Windows 10/11
