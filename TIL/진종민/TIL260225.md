# TIL — 2026-02-25 (Text-to-SQL LoRA 파인튜닝)

## 오늘 한 일 요약
- Text-to-SQL 데이터셋을 불러오고 전처리한 뒤 LoRA로 파인튜닝하는 흐름을 정리했다.
- 학습 후 추론 및 평가 파이프라인을 확인했다.

## 핵심 배움
- LoRA는 적은 파라미터만 학습해 원래 모델 성능을 크게 훼손하지 않고 특화시킨다.
- Text-to-SQL은 전처리 전략과 토크나이징이 성능에 직접 영향을 준다.
- 평가 스크립트로 execution accuracy를 확인할 수 있다.

## 진행 순서
- `uv sync`로 가상환경과 의존성을 설치했다.
- `PEFT_Text2SQL.ipynb`에서 학습 셀을 순서대로 실행했다.
- `python inference.py`로 예측 결과를 생성했다.
- `test-suite-sql-eval-master`에서 평가 스크립트를 실행했다.

## 주요 파일/폴더
- `PEFT_Text2SQL.ipynb`: LoRA 학습 노트북
- `inference.py`: 추론 스크립트
- `data/train.csv`, `data/validation.csv`: 학습/검증 데이터
- `test-suite-sql-eval-master/`: 평가 코드와 예시
- `outputs/`: 학습된 모델 저장 위치

## 사용한 명령어
- `uv sync`
- 필요 시 `uv add numpy==2.1.0 pandas==2.2.3 peft==0.17.1 torch==2.9.0 transformers==4.56.0 accelerate==1.10.1 datasets==4.0.0 trl==0.22.2 sqlparse==0.5.5 nltk==3.9.2 jupyter==1.1.1 ipykernel==7.1.0`
- `python -c "import nltk; nltk.download('punkt_tab')"`
- `cd test-suite-sql-eval-master ; python evaluation.py --gold evaluation_examples/gold.txt --pred evaluation_examples/predict.txt --db database/ --etype exec --plug_value`

## 트러블슈팅 메모
- 평가 전 `database/`에 DB 파일이 있어야 한다.
- Windows PowerShell에서는 세미콜론 구분으로 `cd ... ; python ...` 형태를 사용했다.

## 환경 요약
- Python 3.13
- CUDA 12.9.1
- GPU VRAM 6GB+
- Windows 10/11
