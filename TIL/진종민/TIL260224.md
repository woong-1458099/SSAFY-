# TIL — 2026-02-24 (LLM 기반 Pseudo Labeling으로 데이터 생성)

## 오늘 한 일 요약
- LLM(Teacher)로 정답 라벨을 생성해 Pseudo Labeling 데이터셋을 만드는 흐름을 이해했다.
- Upstage Solar API 키를 설정하고 노트북에서 데이터 생성 파이프라인을 실행했다.

## 핵심 배움
- Knowledge Distillation에서 Pseudo Labeling은 정답이 없는 데이터를 Teacher가 라벨링해 Student 학습 데이터로 만든다.
- 프롬프트 설계가 생성 품질을 좌우한다.
- 로컬에서 큰 모델을 돌리지 않고 API 호출로 데이터 생성이 가능하다.

## 진행 순서
- `.env.example`를 참고해 `.env`에 `UPSTAGE_API_KEY`를 설정했다.
- `uv sync`로 가상환경과 의존성을 설치했다.
- `LLM_기반_데이터_생성.ipynb` 노트북 셀을 순서대로 실행했다.

## 주요 파일/폴더
- `LLM_기반_데이터_생성.ipynb`: 데이터 생성 실습 노트북
- `data/train.csv`: 입력 데이터
- `assets/`: 문서용 이미지
- `pyproject.toml`: 의존성 관리

## 사용한 명령어
- `uv sync`
- 필요 시 `uv add dotenv==0.9.9 openai==1.100.0 pandas==2.2.3 jupyter==1.1.1 ipykernel==7.1.0`

## 트러블슈팅 메모
- `uv`가 인식되지 않으면 PATH 갱신 후 PowerShell 재시작이 필요할 수 있다.
- 크레딧 소모가 빠르면 더 가벼운 모델로 전환하는 것이 안전하다.

## 환경 요약
- Python 3.13
- Windows 10/11
- RAM 8GB+
