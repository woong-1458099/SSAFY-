\# 📝 \[TIL] 2026.03.06: 예외 처리(Exception Handling)의 방법과 활용



\*\*학습 일자\*\*: 2026.03.06



\## 1. 예외 처리란?

\* \*\*정의\*\*: 프로그래머가 예기치 못한 예외(에러)가 발생했을 때, 이에 미리 대처하는 코드를 작성하여 프로그램이 비정상 종료되는 것을 막는 방법론입니다.

\* \*\*목적\*\*: 예외 처리를 하는 가장 중요한 이유는 프로그램의 안정성을 유지하고 서비스의 흐름을 끊기지 않게 보호하기 위함입니다.



\## 2. 예외 처리가 필요한 경우 (사용 시기)

다음과 같이 프로그래머가 예상치 못한 에러가 발생할 경우, 이 상황을 대체하고 대처할 수 있는 코드가 반드시 필요합니다.

\* 파일이 없거나 잘못된 경로의 파일명을 프로그램 내에서 접근하려고 할 때

\* 프로그래머의 실수로 배열의 인덱스 범위를 넘어가버리는(Out of Bounds) 코드가 실행될 때

\* 수학적 오류, 예를 들어 나눗셈을 할 때 `N / 0`과 같은 연산을 시도할 때

\* 기타 네트워크 통신 오류나 예상치 못한 런타임 에러 상황



\## 3. 예외 처리 방법 및 예시



\### 3.1. Throw 또는 Return 문 사용 (Early Return)

\* \*\*개념\*\*: 함수 도입부 등에서 에러를 먼저 체크한 후, 조건에 맞지 않으면 본 코드가 실행되기 전에 함수를 바로 빠져나가는 방식입니다. (자바스크립트 등에서 자주 사용)

\* \*\*특징\*\*: 이러한 방식을 흔히 \*\*Early Return(빠른 반환)\*\*이라고 부릅니다. 본격적인 로직 작성 전에 에러를 먼저 체크하고 거르는 코딩 습관을 들이는 것이 좋습니다.



\### 3.2. Try - Catch 문 사용

\* \*\*개념\*\*: 에러가 발생할 가능성이 있는 코드를 `try` 블록 안에 넣고, 예외가 발생했을 때 프로그램이 종료되는 대신 `catch` (또는 파이썬의 `except`) 블록에서 예외 상황을 처리하는 방식입니다.

\* \*\*특징\*\*: 치명적인 예외가 발생했을 때 다른 대체 코드를 실행하도록 하여 프로그램을 안전하게 계속 유지시킬 수 있습니다.



\### 3.3. Throw와 Try - Catch 혼용 사용 및 Call Stack

\* \*\*개념\*\*: 예외 상황에서 `throw`가 발생되면, 해당 에러 인자를 받아줄 곳을 찾아 \*\*Call Stack (호출 함수 스택)\*\*을 거슬러 올라가며 바로 위의 호출 함수에게 에러를 알립니다. (C++ 등에서 자주 사용)

\* \*\*특징\*\*: 함수가 중첩되어 호출된 상황(예: 1 -> 2 -> 3 순서로 호출)에서도 Call Stack의 동작 원리에 따라 호출의 역순으로 에러를 전파하여 적절한 위치에서 예외를 처리할 수 있습니다.



\### 3.4. 예외 처리와 로깅(Logging)

\* \*\*추가 팁\*\*: 단순한 예외 처리 코드를 넘어 \*\*로그(Log)\*\* 출력 등과 함께 사용하면, 에러가 발생한 지점과 원인을 훨씬 빠르고 정확하게 파악할 수 있어 유지보수에 큰 도움이 됩니다.
