# TIL: 메모리에 대한 이해와 주의점

**학습 일자**: 2026.02.25

## 1. 컴퓨터에서 메모리란?
* 프로그램에서 데이터를 저장할 수 있는 공간입니다.
* 프로그램 실행 시 OS에서 자동으로 일정 영역을 할당해 줍니다.
* 프로그래머라면 반드시 제대로 이해해야 되는 중요한 자원입니다.

## 2. 메모리의 종류와 할당 방식
메모리는 크게 코드, 데이터, 힙, 스택 영역으로 나뉩니다.
* **코드 영역**: 실행할 프로그램의 코드가 저장됩니다.
* **데이터 영역**: 전역 변수와 정적 변수가 저장됩니다.
* **힙 영역**: 사용자의 동적 할당이 이루어지며, 런 타임(Run time)에 크기가 결정됩니다. (예: `class A = new class();`)
* **스택 영역**: 지역 변수와 매개변수가 저장되며, 컴파일 타임(Compile time)에 크기가 결정됩니다. (예: `class B;`)

## 3. 메모리가 중요한 이유
* 메모리는 무한하지 않기 때문에 반드시 관리가 필요하며, 프로그래머가 직접 관리의 주체가 되어야 합니다.
* 프로그램이 종료되면, 할당된 모든 메모리는 OS에 반환되며 날아가게 됩니다.
* 메모리는 영구적이지 않으므로, 데이터를 나중에도 다시 꺼내 쓰려면 저장 매체에 기록해야 합니다.

## 4. 데이터 무결성과 멀티 스레드
* 메모리에 값을 읽고 쓰는 행위는 반드시 동시에 일어나선 안 됩니다.
* 프로그램에서 동시에 한 메모리를 쓸 경우 데이터가 깨지는 현상이 발생할 수 있습니다.
* 순차적인 프로그램에서는 데이터가 깨질 수 없지만, 병렬적으로 데이터에 접근하는 멀티 스레드(multi thread) 환경에서는 반드시 고려해야 하는 요소입니다.
* 이러한 동시 접근 문제는 `LOCK` / `UNLOCK` 키워드로 안전하게 관리할 수 있습니다.
* 결론적으로 프로그래머는 항상 데이터 무결성에 대해 주의해야 합니다.

## 5. 메모리의 저장 매체 (영구 보존용)
* **파일**: 메모리의 데이터를 프로그램에서 직접 하드 디스크에 기록합니다.
* **데이터베이스**: 메모리의 데이터를 DBMS 등의 관리 시스템을 통해 기록합니다.
* **레지스트리**: 운영체제(OS)의 설정 및 정보가 저장되는 데이터베이스입니다.
