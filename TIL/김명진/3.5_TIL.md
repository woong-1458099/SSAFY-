\# 📝 \[TIL] 2026.03.05: React와 Phaser.js를 활용한 웹 게임 개발 및 멀티플레이 구현



\*\*학습 일자\*\*: 2026.03.05



\## 1. React와 Phaser.js 통합 개발 아키텍처

React와 Phaser.js는 각자의 강점이 뚜렷하기 때문에 역할을 명확히 분리하여 통합하는 것이 핵심입니다.



\* \*\*역할 분담\*\*:

&nbsp;   \* \*\*React\*\*: 웹 브라우저의 DOM을 조작하여 정적인 UI(로그인 화면, 로비, 환경설정, 인벤토리, HUD 등)와 전역 상태를 관리합니다.

&nbsp;   \* \*\*Phaser.js\*\*: WebGL/Canvas 위에서 동작하며 게임 루프, 물리 연산, 애니메이션, 캐릭터 렌더링 등 실제 게임의 핵심 로직을 담당합니다.

\* \*\*연결 방식\*\*: React 컴포넌트 내에서 `useRef`와 `useEffect` 훅을 사용하여 특정 `<div>` 태그에 Phaser의 `Game` 인스턴스를 마운트(Mount)하고 언마운트(Unmount) 시 인스턴스를 소멸시키는 방식으로 구현합니다.



\## 2. 상태 관리 및 컴포넌트 간 통신

React의 생명주기(Lifecycle)와 Phaser의 게임 루프(Update loop)는 완전히 독립적으로 돌아가기 때문에, 둘 사이의 통신 규약이 필요합니다.



\* \*\*이벤트 기반 통신 (Event Emitter)\*\*: Phaser 내부에서 발생한 이벤트(예: 몬스터 처치, 점수 획득)를 Custom Event나 Redux, Zustand 같은 전역 상태 관리 라이브러리를 통해 React로 전달하여 UI를 업데이트합니다.

\* \*\*주의점\*\*: React의 상태(State)가 변경되어 컴포넌트가 리렌더링되더라도, Phaser 캔버스 자체가 불필요하게 다시 초기화되지 않도록 철저히 분리해야 합니다.



\## 3. 실시간 멀티플레이 구현 (Socket.io 활용)

멀티플레이 게임은 HTTP 통신(Stateless)이 아닌, 지속적으로 연결이 유지되는 \*\*WebSocket(소켓 통신)\*\*을 활용하여 구현합니다.



\* \*\*서버 (Node.js + Socket.io)\*\*: 

&nbsp;   \* \*\*권위적인 서버 (Authoritative Server)\*\*: 해킹을 방지하기 위해 이동, 충돌, 피격 등 모든 중요한 물리 연산과 데이터 갱신은 서버에서 처리합니다.

&nbsp;   \* 서버는 클라이언트로부터 입력값을 받고, 계산된 최신 게임 상태(위치, HP 등)를 연결된 모든 유저에게 브로드캐스트(Broadcast)합니다.

\* \*\*클라이언트 (React + Phaser)\*\*:

&nbsp;   \* 유저가 방향키를 누르면 클라이언트는 직접 캐릭터를 이동시키는 대신, "오른쪽 이동 키 누름"이라는 \*\*입력(Input) 패킷만 서버로 전송(Emit)\*\*합니다.

&nbsp;   \* 서버로부터 갱신된 위치 데이터를 수신하면 Phaser의 `update()` 메서드에서 캐릭터 스프라이트의 좌표를 갱신합니다.



\## 4. 멀티플레이 최적화 핵심 기법

네트워크 지연(핑, Ping)으로 인한 불편함을 줄이기 위해 아래와 같은 클라이언트 처리 기법이 필수적입니다.



\* \*\*클라이언트 예측 (Client Prediction)\*\*: 유저가 키를 눌렀을 때 서버의 응답을 기다리지 않고, 클라이언트 화면에서 먼저 캐릭터를 움직이게 처리하여 조작 지연을 없앱니다. (이후 서버 데이터와 오차가 생기면 서버 기준으로 보정합니다.)

\* \*\*보간법 (Interpolation)\*\*: 다른 플레이어의 좌표 데이터를 수신할 때 뚝뚝 끊겨서 이동하는(Teleport) 현상을 막기 위해, 이전 좌표와 현재 좌표 사이를 부드럽게 이어주는 연산을 수행합니다.



\## 5. 핵심 요약 (결론)

1\. \*\*역할 분리\*\*: UI와 데이터 로직은 React가, 그래픽과 물리 연산은 Phaser가 담당하게 설계합니다.

2\. \*\*소켓 통신\*\*: 멀티플레이는 양방향 통신인 웹소켓을 사용하여 서버와 클라이언트 간의 패킷을 실시간으로 주고받습니다.

3\. \*\*서버 중심 구조\*\*: 중요한 데이터 연산은 반드시 서버에서 처리(Authoritative Server)하여 데이터 무결성을 지키고 클라이언트 예측 및 보간법으로 UX를 개선합니다.
